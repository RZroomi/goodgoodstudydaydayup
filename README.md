1. FreeRTOS 是什么？和裸机、Linux 区别
FreeRTOS：轻量级开源实时操作系统，专为单片机 MCU 设计，占用资源极小，提供多任务调度、同步通信组件。
和裸机对比：
裸机：单循环，只能顺序执行，高实时任务容易被阻塞；
FreeRTOS：多任务并发，高优先级任务可抢占 CPU，实时性更强。
和 Linux 对比：
Linux：软实时、内存管理复杂、占用 Flash/RAM 大，适合带 MMU 芯片；
FreeRTOS：硬实时、无 MMU、几 KB 内存就能跑，适合 STM32、ESP32 等单片机。
2. 硬实时、软实时，FreeRTOS 属于哪类
软实时：超时不会系统崩溃，只是业务卡顿（Linux）；
硬实时：必须在规定时限内完成响应，超时会功能失效、设备失控；
FreeRTOS 属于硬实时操作系统。
3. FreeRTOS 七大核心组件作用
任务 Task：独立运行的业务流程，每个任务有独立栈、独立优先级；
队列 Queue：任务 / 中断间传递数据，FIFO 缓存，支持结构体；
信号量 Semaphore：二值信号量用于同步，计数信号量用于资源计数；
互斥量 Mutex：保护共享资源，解决多任务竞争，自带优先级继承；
事件组 EventGroup：按位标记事件，一个任务等待多个触发条件；
软件定时器 Timer：基于系统节拍的延时定时器，回调统一由定时器任务执行；
任务通知 TaskNotify：轻量级通信，替代信号量，开销更低，仅支持一对一通信。
4. 任务四种状态 + 流转
四种状态：就绪、运行、阻塞、挂起
运行：当前占用 CPU 正在执行；
就绪：条件满足，等待调度器分配 CPU；
阻塞：主动等待延时、信号量、队列、事件，让出 CPU；
挂起：被手动 Suspend 暂停，不会参与调度，唤醒后回到就绪。
流转核心：
运行→阻塞：调用 vTaskDelay、xSemaphoreTake 等阻塞 API；
阻塞→就绪：延时到 / 拿到信号量 / 收到消息；
运行→就绪：高优先级任务就绪抢占，或同优先级时间片轮转；
就绪 / 运行→挂起：vTaskSuspend；挂起→就绪：vTaskResume。
5. 任务优先级规则
数值越大，优先级越高；0 为最低优先级，空闲任务固定优先级 0；
高优先级任务就绪会立即抢占低优先级任务；
同优先级任务：开启时间片轮转后轮流执行；关闭则先创建的任务一直运行。
6. 时间片轮转调度
同一优先级多个任务平分 CPU 时间，每个任务运行固定节拍时长自动让出 CPU。
开启宏：configUSE_TIME_SLICING = 1。
7. 任务栈作用、栈溢出现象、检测方法
栈作用：保存任务局部变量、函数调用栈、中断上下文寄存器，每个任务独立栈；
栈溢出现象：程序死机、数组乱码、变量异常、HardFault、任务莫名卡死；
两种检测：
栈水印（高水位线）：uxTaskGetStackHighWaterMark()，查看剩余最小栈空间；
栈栅栏：栈末尾填充固定标记，运行时校验是否被改写。
8. 系统启动流程 & 空闲任务
启动流程
main () → 硬件初始化 → xTaskCreate 创建业务任务 → vTaskStartScheduler ()
调度器内部：创建空闲任务、创建定时器任务、开启 Systick 滴答中断、启动第一个最高优先级就绪任务。
空闲任务
系统自动创建，优先级 0，无其他任务可运行时执行；
作用：释放被删除任务的内存、支持低功耗休眠钩子；
不能手动删除，删除会直接系统崩溃。
9. 空闲钩子、定时器钩子
空闲任务钩子 vApplicationIdleHook：空闲循环内执行，适合低功耗、后台巡检；禁止调用阻塞 API；
定时器钩子 vApplicationTimerHook：每次软件定时器节拍触发，用于定时统计、日志打点。
10. configTICK_RATE_HZ 系统节拍
定义每秒 Systick 中断次数，如 100 代表 10ms 一个系统节拍；
所有延时、阻塞等待、软件定时器都基于这个节拍计时；
数值越大计时精度越高，但中断开销变大。
